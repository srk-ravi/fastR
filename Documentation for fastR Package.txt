#' Hybrid PMEST + OCLC FAST Text Classification
#'
#' Performs automated subject classification combining Ranganathan's PMEST 
#' theoretical framework with OCLC FAST authority-controlled terminology.
#'
#' @param title Character string containing the text or title to classify.
#'   Should be descriptive and substantive for optimal classification results.
#' @param domain Character string specifying the classification domain. 
#'   Currently supported: \code{"management"} and \code{"chemistry"}.
#'   Additional domains can be added through taxonomy extension.
#' @param use_fast_integration Logical value indicating whether to integrate
#'   OCLC FAST 9 facets with PMEST 5 facets. Default is \code{TRUE}.
#'   Set to \code{FALSE} for PMEST-only classification.
#' @param confidence_threshold Numeric value (0-1) for minimum confidence
#'   score required for term acceptance. Default is \code{0.5}.
#'   Higher values increase precision but may reduce recall.
#'
#' @return A structured list containing:
#' \describe{
#'   \item{text}{Original input text}
#'   \item{domain}{Classification domain used}
#'   \item{pmest_classification}{List of PMEST facet assignments:
#'     \itemize{
#'       \item \code{personality}: Core subjects, entities, organizations
#'       \item \code{matter}: Materials, properties, resources
#'       \item \code{energy}: Processes, operations, activities  
#'       \item \code{space}: Geographic, organizational, conceptual contexts
#'       \item \code{time}: Temporal periods, chronological aspects
#'     }
#'   }
#'   \item{fast_classification}{List of OCLC FAST 9 facet assignments:
#'     \itemize{
#'       \item \code{topics}: Subject concepts and themes
#'       \item \code{geographic_names}: Places, locations, regions
#'       \item \code{personal_names}: Individual people as subjects
#'       \item \code{corporate_names}: Organizations, institutions
#'       \item \code{form_genre}: Document types, formats
#'       \item \code{time_periods}: Chronological periods, eras
#'       \item \code{titles}: Referenced works, publications
#'       \item \code{events}: Historical events, occurrences
#'       \item \code{meeting_names}: Conferences, gatherings
#'     }
#'   }
#'   \item{integrated_analysis}{Cross-validation between frameworks:
#'     \itemize{
#'       \item \code{cross_validation}: Terms found in both frameworks
#'       \item \code{complementary_assignments}: Unique framework contributions
#'       \item \code{integration_strength}: Numerical integration score
#'     }
#'   }
#'   \item{confidence_scores}{Confidence metrics for classification quality}
#'   \item{metadata}{Processing information, timestamps, assignment counts}
#' }
#'
#' @details
#' The classification process involves several stages:
#' \enumerate{
#'   \item \strong{Text Preprocessing}: Normalization and tokenization
#'   \item \strong{PMEST Matching}: Theoretical facet term identification
#'   \item \strong{FAST Matching}: Authority-controlled term identification  
#'   \item \strong{Integration Analysis}: Cross-framework validation
#'   \item \strong{Confidence Scoring}: Quality assessment and metadata generation
#' }
#'
#' Classification confidence varies by domain complexity and term specificity.
#' Management domain typically achieves 80-90% confidence, while specialized
#' domains like chemistry may show 50-70% confidence due to technical terminology.
#'
#' @section Performance Metrics:
#' Based on testing with diverse academic content:
#' \itemize{
#'   \item Management Domain: 87% average confidence, 100% classification rate
#'   \item Chemistry Domain: 57.7% average confidence, 100% classification rate
#'   \item Cross-Framework Integration: Successful validation in 80%+ of cases
#' }
#'
#' @examples
#' # Basic classification - Management domain
#' result1 <- classify_fast_pmest_hybrid(
#'   "Strategic Leadership in Digital Organizations", 
#'   "management"
#' )
#' print(result1$pmest_classification)
#' print(result1$metadata$overall_confidence)
#'
#' # Chemistry domain classification
#' result2 <- classify_fast_pmest_hybrid(
#'   "Chemical Synthesis of Organic Polymers",
#'   "chemistry"
#' )
#' print(result2$fast_classification$topics)
#'
#' # PMEST-only classification (no FAST integration)
#' result3 <- classify_fast_pmest_hybrid(
#'   "Innovation Management in Technology Companies",
#'   "management",
#'   use_fast_integration = FALSE
#' )
#'
#' # High-confidence classification
#' result4 <- classify_fast_pmest_hybrid(
#'   "Molecular Dynamics Simulations", 
#'   "chemistry",
#'   confidence_threshold = 0.8
#' )

#' Enhanced Multi-Domain Corpus Classification Analysis
#'
#' Performs comprehensive subject classification analysis on document corpora
#' spanning multiple academic domains, utilizing integrated PMEST + OCLC FAST
#' frameworks for systematic faceted analysis.
#'
#' @param corpus_data A data.frame containing document metadata with at least
#'   title and domain information. Additional metadata columns are preserved
#'   in output for reference and analysis.
#' @param title_column Character string specifying the name of the column
#'   in \code{corpus_data} that contains document titles or text to classify.
#' @param domain_column Character string specifying the name of the column
#'   in \code{corpus_data} that contains domain labels for each document.
#' @param domains Character vector specifying which domains to include in
#'   the analysis. Default is \code{c("management", "chemistry")}. 
#'   Documents with domains not in this list are excluded.
#' @param use_fast_integration Logical indicating whether to integrate
#'   OCLC FAST 9 facets with PMEST 5 facets. Default is \code{TRUE}.
#' @param progress Logical indicating whether to display progress messages
#'   during processing. Default is \code{TRUE}. Useful for large corpora.
#'
#' @return A list object of class \code{"fast_pmest_enhanced_results"} containing:
#' \describe{
#'   \item{[domain_name]}{For each analyzed domain, a list of individual
#'     classification results from \code{\link{classify_fast_pmest_hybrid}}}
#'   \item{Attributes:}{
#'     \itemize{
#'       \item \code{total_titles}: Total number of documents processed
#'       \item \code{domains_analyzed}: Vector of domains included in analysis  
#'       \item \code{fast_integration_used}: Whether FAST integration was applied
#'       \item \code{analysis_timestamp}: Processing completion timestamp
#'       \item \code{framework}: Description of classification frameworks used
#'     }
#'   }
#' }
#'
#' @details
#' The corpus analysis workflow includes:
#' \enumerate{
#'   \item \strong{Data Validation}: Verify required columns and data integrity
#'   \item \strong{Domain Filtering}: Process only specified domains  
#'   \item \strong{Individual Classification}: Apply hybrid classification to each title
#'   \item \strong{Results Aggregation}: Compile domain-specific result sets
#'   \item \strong{Metadata Generation}: Create comprehensive processing metadata
#' }
#'
#' Large corpora (>1000 documents) may require significant processing time.
#' Progress reporting helps monitor analysis completion. Results are optimized
#' for subsequent statistical analysis and export operations.
#'
#' @section Memory Considerations:
#' For very large corpora (>10,000 documents), consider:
#' \itemize{
#'   \item Processing in batches by domain or date range
#'   \item Increasing R memory limits: \code{memory.limit(size = 8000)}
#'   \item Using \code{progress = TRUE} to monitor processing status
#' }
#'
#' @examples
#' # Create sample corpus data
#' corpus <- data.frame(
#'   Title = c(
#'     "Strategic Management in Digital Organizations",
#'     "Chemical Synthesis of Pharmaceutical Compounds",
#'     "Leadership Development and Innovation",
#'     "Molecular Dynamics in Biological Systems"
#'   ),
#'   Domain = c("management", "chemistry", "management", "chemistry"),
#'   Year = c(2023, 2022, 2024, 2023),
#'   Author = c("Smith, J.", "Lee, K.", "Brown, M.", "Taylor, R.")
#' )
#'
#' # Comprehensive analysis with FAST integration
#' results <- analyze_multidomain_corpus_enhanced(
#'   corpus_data = corpus,
#'   title_column = "Title", 
#'   domain_column = "Domain",
#'   domains = c("management", "chemistry"),
#'   progress = TRUE
#' )
#'
#' # Examine analysis metadata
#' print(attributes(results))
#'
#' # Access domain-specific results  
#' management_results <- results$management
#' chemistry_results <- results$chemistry
#'
#' # PMEST-only analysis (no FAST integration)
#' pmest_only_results <- analyze_multidomain_corpus_enhanced(
#'   corpus, "Title", "Domain",
#'   use_fast_integration = FALSE
#' )
#'
#' @author Ravikumar S
#' @seealso 
#' \code{\link{classify_fast_pmest_hybrid}} for individual classification,

## 🎯 Framework Overview

### PMEST 5 Facets (Ranganathan)
- **P**ersonality: Core subjects, entities, organizations
- **M**atter: Materials, properties, resources  
- **E**nergy: Processes, operations, activities
- **S**pace: Geographic locations, organizational contexts
- **T**ime: Temporal periods, chronological aspects

### OCLC FAST 9 Facets
1. **Topics**: Subject concepts and themes
2. **Geographic Names**: Places, locations, regions
3. **Personal Names**: Individual people as subjects
4. **Corporate Names**: Organizations, institutions
5. **Form/Genre**: Types of materials, formats
6. **Time Periods**: Chronological periods, eras
7. **Titles**: Referenced works, publications
8. **Events**: Historical events, occurrences
9. **Meeting Names**: Conferences, gatherings

- Use **descriptive titles** with substantive content
- Include **domain-specific terminology** when available  
- Avoid **overly abbreviated** or **cryptic** titles
- Consider **title + abstract** combinations for comprehensive analysis

### Batch Processing

For large corpora, consider batch processing:

Process large corpus in batches
large_corpus <- read.csv("large_corpus.csv")

Split into manageable batches
batch_size <- 1000
n_batches <- ceiling(nrow(large_corpus) / batch_size)

batch_results <- list()
for (i in 1:n_batches) {
start_row <- (i-1) * batch_size + 1
end_row <- min(i * batch_size, nrow(large_corpus))

batch_data <- large_corpus[start_row:end_row, ]

batch_results[[i]] <- analyze_multidomain_corpus_enhanced(
batch_data, "Title", "Domain",
progress = TRUE
)

cat("Completed batch", i, "of", n_batches, "\n")
}
